## 3. 確認事項(回答をお願いします)

**Q1. デザインの方向性(トーン)**
マスタメンテナンスという業務ツールとしての UI トーンをどれにしますか?

- A) 業務システム標準 — 情報密度高め、装飾控えめ、テーブル中心の実務的デザイン(推奨)
- B) モダン SaaS 風 — 余白広め、カード多用、柔らかい印象
- C) ミニマル/フラット — 線と余白のみで構成、色は最小限
- D) その他(自由記述)

[Answer]: A(業務システム標準 — 高密度・実務的)

**Q2. プライマリカラー(基調色)**

- A) ブルー系(#2563EB 前後)— 業務系の定番、ライト/ダークとも視認性良好(推奨)
- B) ティール/シアン系(#0D9488 前後)
- C) グリーン系(#16A34A 前後)
- D) モノクロ基調 + 最小限のアクセント色
- E) その他(色指定)

[Answer]: A(ブルー系 #2563EB 前後)

**Q3. コンポーネントのスタイリング実装方式**
トークンは CSS カスタムプロパティで確定(D-10)。コンポーネント側のスタイル記述は?

- A) CSS Modules(.module.css)+ カスタムプロパティ参照 — 追加依存なし、Vite 標準サポート、スコープ安全(推奨)
- B) プレーン CSS + BEM 命名 — 単純だがスコープ管理は規約頼み
- C) CSS-in-TS(vanilla-extract 等)— 型安全だが依存とビルド設定が増える
- D) その他

[Answer]: A(CSS Modules + カスタムプロパティ)

**Q4. フォント**

- A) システムフォントスタック(OS 標準: Hiragino/Yu Gothic/Segoe UI 等)— 外部依存なし、CSP 影響なし、オフライン安全(推奨)
- B) セルフホスト Web フォント(Noto Sans JP 等を WAR に同梱)— 全環境で同一の見た目になるがサイズ増
- C) その他

[Answer]: B(セルフホスト Web フォント — Noto Sans JP を WAR に同梱)

**Q5. ベースサイズ・情報密度**

- A) ベース 14px・高密度 — 業務テーブルの一覧性重視(推奨)
- B) ベース 16px・標準密度 — 可読性重視
- C) 密度トークンで切替可能にする(コンパクト/標準)— 柔軟だが実装・確認コスト増
- D) その他

[Answer]: A(ベース 14px・高密度)

**Q6. i18n ライブラリ(言語切替 UI の先行実装で使用)**
モックは実装基盤になるため、②で導入するライブラリがそのまま本実装(③以降)に引き継がれます。

- A) react-i18next — デファクト、辞書 JSON 分離、型付けも可能(推奨)
- B) 自前の軽量実装(Context + 辞書オブジェクト)— 依存ゼロだが複数形・補間等を自作
- C) LinguiJS — コンパイル型でバンドル小、ただし採用実績は i18next より少ない
- D) その他

[Answer]: A(react-i18next)
