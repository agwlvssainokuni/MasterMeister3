# 代表画面仕様(モック対象)— unit-02-design-system

**作成日**: 2026-07-19
**位置づけ**: モック承認ゲート(D-08)で確認する代表画面 4 系統の仕様。本実装(③〜⑥)の画面はこの画面パターンを踏襲する。モックは静的サンプルデータで動作し、API 呼び出しは行わない。

## 1. ログイン画面(→ ③で本実装)

- **レイアウト**: 中央寄せカード(AppShell 外)。アプリ名、メールアドレス(TextInput)、パスワード(PasswordInput)、ログイン(Button primary・loading 対応)、登録申請へのリンク。カード右上またはフッターに LanguageSwitcher / ThemeToggle
- **状態**: 入力エラー(FormField error)、認証失敗(Alert danger — メッセージは失敗理由を特定しない文言)、ローディング
- **モックでの確認点**: フォーム密度、エラー表示、両テーマ・両言語での見え方

## 2. 管理系一覧画面(代表: ユーザ管理 → ③で本実装)

- **レイアウト**: AppShell + ページタイトル + ツールバー(SearchInput、状態フィルタ Select、主操作 Button)+ Table + Pagination
- **Table 列例**: メールアドレス / 表示名 / 状態(Badge: 承認待ち=warning、有効=success、無効=neutral)/ 登録日時 / 操作(IconButton: 承認・無効化 — ConfirmDialog 連携)
- **状態**: ローディング、0 件(EmptyState + フィルタ解除アクション)、操作成功/失敗(Toast)
- **モックでの確認点**: 一覧画面の標準パターン(接続管理・権限管理・監査閲覧など④⑥の画面もこのパターンを踏襲)

## 3. レコード一覧・編集画面(→ ⑤で本実装)

- **レイアウト**: AppShell + テーブル選択コンテキスト(接続/スキーマ/テーブル名 + 実効権限 Badge)+ ツールバー(検索・フィルタ・新規行追加 Button・一括反映バー)+ 編集可能 Table + Pagination
- **セル表現**(⑤の編集バッファ表示の土台):
  - 編集済みセル: primary-subtle 背景 + 左ボーダー
  - バリデーションエラーセル: danger-subtle 背景(Tooltip でエラー内容)
  - READ 専用カラム: 通常表示・編集不可(NONE カラムはそもそも列非表示 — D-18/D-21 前提)
  - 新規行 / 削除予定行: 行単位の背景 + 行頭 Badge
- **一括反映バー**: 変更件数(追加 n / 更新 n / 削除 n)+ 反映(primary)+ 破棄(ghost)。反映は ConfirmDialog(オールオアナッシング — US-027 の文言)
- **状態**: ローディング、0 件、反映成功/失敗(Toast。失敗時はエラーセルへ誘導)
- **モックでの確認点**: 高密度テーブルでの編集状態の視認性(両テーマで)

## 4. クエリ実行画面(→ ⑥で本実装)

- **レイアウト**: AppShell + 上部 Tabs(ビルダー / SQL)+ 実行ツールバー(実行 Button、保存、パラメータ入力欄)+ 下部結果 Table(結果セットメタデータ由来の列)+ 実行情報(件数・実行時間)
- **ビルダータブ**: テーブル選択、カラム選択(Checkbox 群)、WHERE 条件行(カラム Select + 演算子 Select + 値 TextInput の繰り返し + 追加/削除)、ORDER BY
- **SQL タブ**: CodeBlock(編集可能領域 + 等幅フォント)。ビルダー⇄SQL の相互変換は⑥の実装(モックは見た目のみ)
- **状態**: 実行中(結果領域 Spinner)、エラー(Alert danger: SQL エラーメッセージ)、0 件(EmptyState)
- **モックでの確認点**: 2 ペイン構成の使い勝手、等幅フォント表示、結果テーブルの密度

## 5. テーマ切替・言語切替の機能仕様(US-047/048 前段)

②で UI とロジックを実装し、③で「ログインユーザ設定としての保存」へ拡張する(2 段構え)。

- **テーマ(US-048 前段)**:
  - 3 値: `light` / `dark` / `system`(既定 `system` — OS 設定連動、D-10)
  - ThemeToggle(Header 内 Dropdown): 3 値から選択。選択は `localStorage("mm.theme")` に保存
  - `system` 時は `prefers-color-scheme` の変化に即時追従。手動選択時は `data-theme` を固定
  - 切替は再読み込みなしで即時反映(CSS カスタムプロパティの差し替え)
- **言語(US-047 前段)**:
  - 2 値: `ja` / `en`(D-09)。既定はブラウザ言語(`navigator.language` が ja 系なら ja、それ以外 en)
  - LanguageSwitcher(Header 内 Dropdown): 選択は `localStorage("mm.lang")` に保存し、react-i18next の `changeLanguage` で即時反映
  - `<html lang>` 属性も同期する
- **③への引き継ぎ**: ログイン後はユーザ設定(サーバ保存)を正とし、localStorage は未ログイン時・初期表示のちらつき防止に使う。この統合は③のスコープ

## 6. アクセシビリティ・表示の共通事項

- 全画面で両テーマ × 両言語の 4 通り表示をモックカタログ上で切替確認できること
- 対応ブラウザ幅: 1024px 以上を主対象(業務利用)。それ未満は横スクロールで成立させる(専用モバイルレイアウトは作らない)
- 日時表示は `YYYY-MM-DD HH:mm:ss`(ロケールによらず固定 — 監査・データ確認用途のため)
