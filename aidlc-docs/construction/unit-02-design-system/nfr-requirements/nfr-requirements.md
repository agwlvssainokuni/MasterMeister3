# NFR Requirements — unit-02-design-system(簡略)

**作成日**: 2026-07-19

| ID | NFR | 内容 | 検証方法 |
|---|---|---|---|
| NFR-U2-01 | CSP 整合(H-06) | デザインシステム・モックは**外部オリジン参照ゼロ**(フォント・CSS・JS・画像すべて self ホスト)。CSP ヘッダー本則の導入は③のセキュリティ設定スコープだが、②の資産が `font-src 'self'` / `style-src 'self'` で成立することを先行担保 | `vite build` 成果物(dist/)に `http(s)://` の外部アセット参照がないことを検査 |
| NFR-U2-02 | モック本番除外(NFR-06) | `/mock` ルートとサンプルデータは `import.meta.env.DEV` ガード + dynamic import により本番バンドルへ含めない | dist/ に mock チャンク・カタログ文言・サンプルデータが含まれないことを文字列検索で検査(Code Generation の DoD) |
| NFR-U2-03 | フォント配信性能 | woff2 のみ・unicode-range サブセット分割(必要グリフ範囲のみ遅延ロード)・`font-display: swap`。初回表示でフォント全量をダウンロードしない | dev サーバでネットワークタブ確認(日本語ページ初期表示で読み込む woff2 が全体の一部であること) |
| NFR-U2-04 | アクセシビリティ | テキスト・操作要素は WCAG AA コントラスト(両テーマ)。全操作要素キーボード到達可能 + フォーカスリング表示。モーダルはフォーカストラップ | トークンカタログのコントラスト確認欄 + コンポーネントカタログでのキーボード操作確認(モック承認ゲートの確認観点に含める) |
| NFR-U2-05 | バンドル影響 | design-system は named export で tree-shakable に構成。②完了時点の本番 JS バンドル(gzip)は目安 300KB 以下(React + router + i18n + コンポーネント。フォントは別カウント) | `vite build` のサイズレポート確認 |

## 適用外(このユニットでは対象外)

- スケーラビリティ・可用性・スループット: 静的資産のみで①の配信方式(WAR 同梱)に従属するため個別要件なし
- 認証・認可・監査: ③以降のスコープ(モックは静的データのみで実データを扱わない)
