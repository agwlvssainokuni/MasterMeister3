## 2. プロジェクト概要

- RDBMS に格納されたマスタデータをメンテナンスする Web アプリケーション
- **開発体制**: 単独開発者による初期開発
- **開発方針**: MVP ファーストで段階的に機能を追加
- **実装優先順位**: ユーザ管理 → 対象 RDBMS セットアップ → アクセス制御 → データ表示(ワークフロー上流から)

## 4. 機能要件(サマリーと ID 付与)

詳細仕様はすべて project-overview.md の該当節を正とする。

| ID | 機能 | 概要 | 詳細 |
|---|---|---|---|
| FR-01 | ユーザ登録 | メールアドレス先行の 2 段階登録、管理者承認ワークフロー、初期管理者ブートストラップ、列挙攻撃対策 | §5.1 |
| FR-02 | 対象 RDBMS 接続管理 | 複数接続の登録・管理、接続パスワードの可逆暗号化保存 | §5.2, §3 |
| FR-03 | スキーマ取込 | テーブル/ビュー構造(物理名、コメント、型、制約)の読み取りと内部 DB への取込(接続 ID 単位) | §5.2 |
| FR-04 | アクセス制御 | 主権限(NONE/READ/UPDATE、カラム>テーブル>スキーマ優先)+ 補助権限(CREATE/DELETE)。グループ合成(許可優先、個別設定が最優先)。実効権限の Caffeine キャッシュと無効化 | §5.2 |
| FR-05 | 権限の YAML エクスポート/インポート | インポートは全置換方式、重複エントリは全体拒否(トランザクション単位) | §5.2 |
| FR-06 | グループ管理 | 作成・改名・削除(カスケード)、所属管理 | §5.2 |
| FR-07 | ユーザ認証 | JWT(アクセス + リフレッシュ)、ローテーション、再利用検知によるトークンファミリ一括失効 | §5.3 |
| FR-08 | マスタメンテナンス | テーブル/ビュー一覧、ページング付きレコード一覧、絞り込み、インライン編集、オールオアナッシングの一括反映(統一 API) | §5.4 |
| FR-09 | クエリビルダー | タブ構成(SELECT〜LIMIT OFFSET)、スキーマ非修飾 SQL 生成、SQL からのリバースエンジニアリング | §5.5 |
| FR-10 | クエリ保存 | 名前付き保存、公開/非公開、スキーマ非依存、作成者のみ編集、retired 化 | §5.6 |
| FR-11 | クエリ実行 | 読み取り専用 SQL のみ、:param パラメータ化、実行時スキーマ指定(許可リスト検証、DB 方言別切替)、ページング、履歴記録 | §5.7 |
| FR-12 | クエリ履歴 | ページング付き一覧、種別表示、絞り込み、他機能への遷移 | §5.8 |
| FR-13 | 監査ログ | 認証・管理操作・データアクセスイベントの記録(内部 DB、構造化)、管理者のみ閲覧 | §6 |

**対象 RDBMS(必須サポート)**: MySQL / MariaDB / PostgreSQL / H2
