## ペルソナ 1: アキラ(システム管理者)

| 項目 | 内容 |
|---|---|
| 役割 | MasterMeister3 の管理者。情報システム部門の担当者 |
| 目標 | 業務部門が安全にマスタデータを扱える環境を、最小の運用工数で維持する |
| 技術リテラシー | 高い。SQL・RDBMS の構造を理解し、接続情報やスキーマの概念に精通 |
| 主な利用シーン | 対象 RDBMS 接続の登録とスキーマ取込 / ユーザ登録の承認・却下 / グループと権限の設定・YAML でのバックアップと移行 / 監査ログの確認 / ロックされたアカウントの解除 |
| フラストレーション | 権限設定の影響範囲が見えないと不安になる。手作業での権限一括設定は間違いが起きやすい。セキュリティ上の異変に気づくのが遅れたくない |

## ペルソナ 2: ユウキ(業務ユーザ)

| 項目 | 内容 |
|---|---|
| 役割 | 業務部門の担当者。マスタデータの参照・修正を日常業務として行う |
| 目標 | 必要なマスタデータを素早く見つけ、安全に修正を反映する。定型的な参照作業は保存クエリで再利用したい |
| 技術リテラシー | 中程度。表計算ソフトは得意。SQL は SELECT 文の読み書きが少しできる程度 |
| 主な利用シーン | テーブルを開いてレコードを絞り込み・編集・反映 / クエリビルダーで参照クエリを組み立てて保存 / 保存クエリの実行と履歴の確認 |
| フラストレーション | 権限がなくて操作できない理由がわからないと困る。誤って他人のデータを壊してしまうことが怖い。複雑な SQL を毎回書きたくない |
